3. Github gör det möjligt för flera personer att arbeta på samma projekt utan att förstöra för varandra. Varje person kan jobba på en sin kopia av projektet som kallas för branches. En branch är en kopia av huvudprojektet som du kan testa olika grejer utan att riskera at förstöra huvudprojektet. När du har gjort dina fungerande ändringar och vill lägga till det i huvudprojektet skickar du en pull request. En pull request innebär att du skickar din version till github där en eller flera personer kontrollerar att dina ändringar är fungerande. Om dessa ändringar funkar på ett bra sätt slår man sedan ihop de med huvudprojektet vilket kallas för en merge.
