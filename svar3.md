## 3. GitHub  

**GitHub** gör det möjligt för flera personer att arbeta på samma projekt utan att förstöra för varandra.  

Varje person kan jobba på sin egen kopia av projektet, som kallas för **branch**.  
En branch är en kopia av huvudprojektet där du kan testa olika saker utan att riskera att förstöra huvudprojektet.  

När du har gjort dina fungerande ändringar och vill lägga till dem i huvudprojektet skickar du en **pull request**.  

En pull request innebär att du skickar din version till GitHub där en eller flera personer kontrollerar att dina ändringar fungerar.  
Om ändringarna är bra slår man sedan ihop dem med huvudprojektet, vilket kallas för en **merge**.
